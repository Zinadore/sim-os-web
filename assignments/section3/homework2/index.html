Homework 2

Homework 2

Points: 50

Due by 11:59 EDT on November 17, 2019

Introduction

In this homework you will be implementing three process scheduling algorithms for a simulated OS called SimOS, then analyzing their results and performance. You will be using Java, Processing, and of course SimOS.

You will be submitting everything through GitHub. Before you begin, be sure to accept the homework on GitHub at https://classroom.github.com/a/ixhcjsUk

Set Up

If you haven't already, visit the Wiki to set up the project in your own computing environment. Set up will include downloading Processing, SimOS and creating the basic infrastructure code for this homework.

In the wiki you will also find links to the Java documentation, SimOS documentation and other resources. They are there for you to use so do not ignore them! You will also find that every Java class mentioned in the Wiki is a link to the matching documentation page.

The rest of this document assumes you have gone through the setup and are somewhat familiar with the framework.

Algorithms

You have to implement and perform analysis on three (3) process scheduling algorithms in total. Those are:

  1. Shortest Job First
  2. Shortest Remaining Time Next
  3. Round Robin
Round Robin requires a parameter to be passed in the constructor. That parameter is the number of time slices each process gets. The constructor should look something like the following: 
1 public PS_RR(int slices) {
2     super();
3     // Store slices somewhere
4 }

Assignment

You must implement the three algorithms supported by SimOS (see above). You are to do this on your own.

Write Up

The last part of this homework is your algorithm analysis. As part of your write-up you must select any two of the three implemented algorithms and compare and contrast them. You do not need to write a novel here (it should be no longer than a paragraph or two), you should have data from the simulation. Base your explanation on that data! If you made any interesting design decisions in your implementations, you can discuss them here.

In addition to the comparison you should select one of the three algorithms and create an input of processes that will expose the algorithm's weaknesses. Include in your write up why your data set does that.

The algorithm can be on of the two discussed previously, or it can be the third one. It is up to you to decide. You can find information on how to create your own datasets here.

Submitting

Each person will have to submit all three algorithms, the sketch file, the analysis and the input data set. Everything should be in the repository you created when you accepted this assignment.

Make sure your analysis is in PDF format. Any other format will not be accepted.

So all in all you need:

  • 1 .pde file (this is the sketch)
  • 3 .java files (these are the algorithms)
  • 1 .pdf file (this is your analysis)
  • 1 .txt file (this is your input set)

To submit make sure your files are in their own directory, then run the following command in that directory (substituting the list of files you need to commit for your_files_go_here and your GitHub username for username, of course):

git init
git add your_files_go_here
git commit
git remote add origin git@github.com:UMBC-CMSC421-FA2019/simos-section-3-username.git
git push -u origin master

git tag hw2
git push origin --tags
