Running the simulation

We are almost done! Now that we have our two algorithms we can run the simulation. But first we need to tell it which algorithms we have implemented.

Back to setup()

At the very top of your sketch, you need to include EnumSet from the standard Java library since we will be using it. You can just add the line:

1 import java.util.EnumSet;

Going back to the sketch and the setup method, we need to let the simulation know which classes we are providing, so it can load them. We need to change our setup method from: 

1 Simulator sim; // <-- Don't forget this line!
2 void setup() {
3     SimulatorOptions opts = new SimulatorOptions();
4     opts.Width = width;
5     opts.Height = height;
6 
7     sim = new Simulator(this, opts);
8 }

to

 1 Simulator sim; // <-- Don't forget this line!
 2 void setup() {
 3     SimulatorOptions opts = new SimulatorOptions();
 4     opts.Width = width;
 5     opts.Height = height;
 6 
 7     opts.ProcessSchedulers = EnumSet.of(
 8         ProcessSchedulerType.FirstComeFirstServed
 9     );
10 
11     opts.MemoryManagers = EnumSet.of(
12         MemoryManagerType.FirstFit
13     );    
14 
15     opts.EnableDarkMode = true;
16 
17     sim = new Simulator(this, opts);
18 }

By providing opts.ProcessSchedulers and opts.MemoryManagers the framework will generate the correct UI for us and load the classes you created. If you wanted to add RoundRobin as well for example you could do it as such: 

1 opts.ProcessSchedulers = EnumSet.of(
2     ProcessSchedulerType.FirstComeFirstServed,
3     ProcessSchedulerType.RoundRobin
4 );

You can also toggle dark mode on and off from the options. Check both modes to see what suits you best !

Pixels!!

Time to see those pixels drawn! Hit the "Play" button in the Processing IDE, select the options you want, and click "Start Simulation". You should see a screen similar to this:

The buttons should be self-explanatory. Forward can be used to step one tick at a time. By default the simulation will try to have one CPU tick per second. However if that is too slow (or you have a lot of processes), you can use the slider to speed things up!
