Basic Setup

With everything installed we can get started with our simulation. In the next few sections, we will create a sketch, setup the simulation then implement some basic algorithms.

Creating the sketch

Sketches are processing's project format. They are written in Java and you can use the integrated IDE to develop your code.

So, startup processing and go to File > Save As to save the sketch wherever you want. I am saving mine at ...\Documents\Sketches\HW2 for example. Next up we will create three functions that processing will call for us. You can always consult the Processing API if you need to. But first we need to import the library we installed earlier.

Importing the library

Go to Sketch > Import Library > Select SimOS. The sketch will update to look like this:

These import lines, will allow you to use the classes that are part of the library.

Sketch Methods

The next three methods are sort of "magic", in the sense that processing will call them on its own throughout its runtime.

settings()

This method can be used to set some parameters before our window shows up. Here we are just settings the resolution of the window. 

1 void settings() {
2     // Width, Height
3     size(800, 900);
4 }

setup()

This method will get called after our window has been created. This is where we create our Simulation and define its options. 

1 Simulator sim; // <-- Don't forget this line!
2 void setup() {
3     SimulatorOptions opts = new SimulatorOptions();
4     opts.Width = width;
5     opts.Height = height;
6 
7     sim = new Simulator(this, opts);
8 }

For now the only options we set are the Width and Height

draw()

This method gets called whenever a new frame needs to be drawn. Processing will try it's best to run at 60 or 30 frames per second depending on your hardware. 

1 void draw() { }

You might be surprised to see the method completely emtpy. But you need it to be there, even if it is empty. If you forget this method the simulation will only draw itself once then stop!

Sanity Check

You can now hit the "Play" button in the top right and see the following screen:

Make sure you can click the various buttons and that they get activated. Do not click Start Simulation though! We need to implement some of these algorithms for the simulation to work!

If the Simulation freezes for any reason, you can just hit the "Stop" button from inside the Processing IDE. You will also see some error messages in the console!
